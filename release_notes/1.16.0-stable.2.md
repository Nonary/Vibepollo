# Vibepollo 1.16.0-stable.2 - 2026-06-10

**Notice:** Vibepollo may trigger false-positive antivirus alerts. Details and workarounds are documented here: https://github.com/Nonary/Vibepollo/issues/59

## Changes

* Added an optional **v2 display helper engine** that can be selected for prerelease soak testing while stable users continue to use the legacy helper by default. The new engine adds a state-machine restore path, shared snapshot handling, bounded verification before capture starts, configuration controls, and focused unit coverage.

## Fixes

* Fixed a virtual display restore loop that could repeatedly reapply display settings after a stream ended and scramble Windows MPO overlay planes.

* Fixed an NVIDIA encoder teardown race that could crash multi-client streams during secure desktop or UAC transitions.

* Fixed a TLS certificate ownership bug that could crash Vibepollo during Moonlight server info polling or TLS handshakes.

* Fixed display restore scheduling and isolated-layout placement so monitor recovery still works after sign-out, reboot, or lock-screen streaming.

* Fixed poisoned virtual-display baselines that could restore an all-virtual layout and disable the user's physical monitor after streaming.

* Fixed client permission denials so launch, resume, cancel, and no-list-permission responses explain which permission is missing and the host logs actionable warnings.
