# Vibepollo 1.16.0-beta.5 - 2026-05-30

Notice: Vibepollo may trigger false-positive antivirus alerts. Details and workarounds are documented here: https://github.com/Nonary/Vibepollo/issues/59

## Changes
- Updated the experimental Vibepollo Display Driver package to libvirtualdisplay 1.2.3, improving adapter selection on multi-GPU systems and preferring 10-bit RGB modes where supported.

## Fixes
- Fixed reported crashes when starting or stopping streams and when UAC prompts triggered display changes by updating the virtual display driver and avoiding swapchain startup stalls during display recreation.
- Fixed transient WebRTC video corruption after dropped encoded frames by waiting for a fresh keyframe before forwarding dependent frames again.
- Fixed paired clients sometimes losing their saved virtual display identity, which could create a fresh Windows monitor and discard monitor-specific settings.
- Fixed session teardown hangs when capture reinitialization was waiting on encoder display references after the capture queue had already stopped.
- Fixed passive virtual display availability checks accidentally repairing or recreating devices outside explicit initialization.
