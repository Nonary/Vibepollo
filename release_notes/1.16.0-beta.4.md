# Vibepollo 1.16.0-beta.4 - 2026-05-29

Notice: Vibepollo may trigger false-positive antivirus alerts. Details and workarounds are documented here: https://github.com/Nonary/Vibepollo/issues/59

## Changes
- Updated the experimental Vibepollo Display Driver package to libvirtualdisplay 1.2.1, adding support for 5120-wide ultrawide temporary displays such as Samsung G9-class clients.

## Fixes
- Fixed remote WebRTC sessions failing before negotiation when browsers advertised extra codecs the server could not parse.
- Fixed bootstrapper upgrades choosing browser-owned Chrome or Edge web app registrations instead of the real Vibepollo install directory.
- Fixed experimental virtual display driver install failures on non-English Windows systems by using a language-neutral User Mode Drivers SID and reporting driver setup problems without blocking the rest of Vibepollo.
- Fixed Web UI launch links when the Web UI is bound to a specific LAN or IPv6 address instead of localhost.
